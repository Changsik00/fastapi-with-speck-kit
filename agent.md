# 에이전트 행동 강령 (`agent.md`)

이 문서는 에이전트의 상호작용 규칙과 운영 모드를 정의합니다. 어떤 프로젝트든 이 파일을 루트 디렉토리에 복사해두면 에이전트가 동일한 규칙을 따르게 됩니다.

## 1. 운영 모드 (Operational Modes)
에이전트는 사용자의 요청을 분석하여 다음 두 가지 모드 중 하나로 분류해야 합니다.

### 모드 A: 표준 SDD (기본값)
- **발동 조건**: 새로운 기능 개발, 복잡한 리팩토링, 아키텍처 변경, 또는 "직접 요청"으로 처리하기엔 너무 복잡해진 경우.
- **프로세스**: 엄격한 **사양 주도 개발(SDD)** 사이클(명세 -> 설계 -> 구현)을 따릅니다.
- **목표**: 높은 신뢰성, 변경 사항의 문서화.

### 모드 B: 직접 요청 (빠른 처리)
- **발동 조건**: 단순 Q&A, 소규모 버그 수정(1-2개 파일), 문서 업데이트, 일회성 스크립트 작성.
- **프로세스**:
  1. **실행 (Execute)**: 변경 사항을 즉시 구현합니다.
  2. **검증 (Verify)**: 테스트를 실행하여 회귀(Regression)가 없는지 확인합니다.
  3. **격상 (Escalate)**: 만약 작업이 복잡해지면 즉시 **모드 A**로 전환합니다.
- **목표**: 사소한 작업의 속도와 효율성.

## 2. 커뮤니케이션 프로토콜 ("수락/취소")
중요한 결정을 내릴 때, 에이전트는 반드시 `BlockedOnUser=true`를 사용하여 사용자의 확인을 받아야 합니다.

### 수락 (Accept) - 진행
- **영어**: `Accept`, `Yes`, `Y`, `Ok`, `Okay`, `Good`, `Go`, `G`, `A`, `1`, `Dd`, `Dz`
- **한국어**: `ㅇㅇ`, `ㅇㅋ`, `어`, `좋아`, `ㄱㄱ`, `진행`

### 취소 (Cancel) - 중단
- **영어**: `Cancel`, `No`, `N`, `Stop`, `Quit`, `Wait`, `X`, `2`, `Ss`
- **한국어**: `ㄴㄴ`, `아니`, `취소`, `멈춰`, `잠깐`

## 3. 핵심 행동 수칙
- **능동성 (Proactive)**: 요구사항이 모호하면 작업을 시작하기 전에 먼저 질문합니다.
- **안전 (Safe)**: 완료를 보고하기 전에 반드시 변경 사항을 검증(테스트/빌드)합니다.
- **문맥 인식 (Context-Aware)**: 변경을 제안하기 전에 프로젝트 구조와 기존 문서를 먼저 파악합니다.
